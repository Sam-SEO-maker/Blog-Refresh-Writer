# Content Writer

Autonomous **multi-site SEO refresh** agent: optimises existing content from
data signals (GSC + DataForSEO), preserving each site's editorial identity.
Data-driven decisions (audit → decision → generation → QC → internal linking).

Current sites: `enseigna`, `superprof-ressources`. The registry is open
(`_shared/config/sites.json`): onboarding a site = `python3 content_writer.py
site init <site-slug>` (skeleton `sites/<site-slug>/` + `sites.json` entry + sparse-checkout),
then fill in the editorial part. See [onboarding/](onboarding/README.md).

## Onboarding (new SEO Managers)

Each site's SEO Manager (ES, UK, US, MX, ID, JP…) clones **only their site** via a
git sparse-checkout: the shared engine + their single `sites/<site-slug>/`, never the other
sites. Full walkthrough: **[onboarding/README.md](onboarding/README.md)**.

Recommended clone (sparse):

```bash
bash onboarding/scripts/setup_sparse.sh <site-slug>   # e.g. es-es-ressources
```

## Installation (maintainer / full clone)

```bash
pip install -r requirements.txt
cp .env.example .env   # then fill in the credentials (DataForSEO, WP, GSC…)
```

## Usage

The CLI is `content_writer.py`. The up-to-date list of groups and commands is
auto-generated by Click:

```bash
python3 content_writer.py --help
python3 content_writer.py <group> --help    # e.g. refresh, batch, audit, site
python3 content_writer.py site list        # catalog of onboardable sites
python3 content_writer.py site init <site-slug>   # onboard a site (skeleton + sparse)
```

Example: `python3 content_writer.py refresh <url> --site enseigna`

## Golden Rule (invariant)

Never reduce a refreshed article's assets: `assets_after ≥ assets_before`
(images, tables, videos, internal **and** external links, including links to competitors).
